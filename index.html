Stage 1:Where it All Begins

Lesson 1: The Basics of the Web and HTML

Web Page

A web page is a text document that is written in the HTML language(heart of the web). The web browsers will then read these documents, interpret, and lastly display them.

Coding

Programmers will write text in a language that a computer can understand. The computer will then follow the instructions written by the programmer.

Computers Are Not Smart

Therefore as programmers we need to write exactly the way a computer understands. This idea is known as writing with the correct syntax.

Programmers Can't Remember Everything

Since there are too many details for us to remember, we can always look it up.Google is our friend.

Vocab

Tag:An HTML tag is always contained withing angled brackets. Most tags have an opening tag and a closing tag. A void tag is one that does not have a closing tag.
Element:An HTML element refers to everything within a set of opening and closing tags. Attribute: This is a property of an HTMl element.

Inline vs Block Element

HTML elements are either inline or block. A block element creates an invisible box around the text.

Lesson 2: Creating a Structered Document with HTML

HTML

Hyper Text Markup Language: the standard markup language used to create web pages.

CSS

Cascading Style Sheets: style sheet language used for describing the look and formatting of a document written in a markup language.

DOM

Document Object Model: every document is organized in a tree structure.The HTML elements can have other elements inside of them. You can draw this relationship like a family tree.

Lesson 3: Adding CSS Style to HTML Structure

Box Sizing

HTML elements are boxes and has many components:

- Because there are so many components to each box, it can be hard to get the size of a box just right.
- There are two techniques you can use to help deal with sizing issues:
Set sizes in terms of percentages rather than pixels.
Set the box-sizing attribute to border-box for every element.
- Different browsers work slightly differnetly. Sometimes causing different broswers to display the same code differently.

Box Positioning

Divs are block elements(as opposed to inline), therefore they take up the entire width of the page by default

Repetition

It's good to avoid repetition when programming. By giving the same style to similar elements, we can quickly modify all of them at once by changing single line in CSS. For example in my code, each lesson was assigned the same class and a specified style that made it easy for it to be in one place.
